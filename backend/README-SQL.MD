1. תחילה הורידו את PostgreSQL 
הורדה: היכנסו לאתר הרשמי של PostgreSQL והורידו את ה-Installer המתאים למערכת ההפעלה שלכם (Windows/macOS).

הרצה: הפעילו את ה-Installer ובחרו את הרכיבים הבאים:

PostgreSQL Server: שרת מסד הנתונים עצמו.

pgAdmin 4: כלי ממשק גרפי (GUI) לניהול בסיס הנתונים (מומלץ).

Command Line Tools: כלי שורת פקודה חיוניים (כגון psql ו-pg_dump).

הגדרת סיסמה: בשלב זה, תתבקשו להגדיר סיסמה למשתמש הניהול הראשי (postgres). שמרו סיסמה זו במקום בטוח.

הגדרה: השאירו את הפורט (Port) על ערך ברירת המחדל 5432.

כנסו לvs code terminal ורשמו
cd "C:\Program Files\PostgreSQL\18\bin"
2. 
התחברות: התחברו לשרת באמצעות הכלי psql והמשתמש הראשי:
psql -U postgres

יופיע להזין סיסמא שבחרנו הזמן ההתקנה שימו לב שלא תראו את הסיסמא עצמה במהלך ההקלדה

3. יצירת מסד נתונים חדש
CREATE DATABASE name of the project



4. יציאה מpsql
\q

5. כניסה למסד נתונים ספיציפי
\c buyforce
זה המסד שנתונים שבו יצרתי את הטבלאות

6. הצגת הטבלאות ממסד הנתונים
\dt

מציג את מבנה העמודות, הסוגים והאינדקסים של טבלת users
\d users

7. מציג רשימה של כל מסדי הנתונים הקיימים בשרת.
\l

8.פקודת SQL סטנדרטית ליצירת טבלה חדשה. 
CREATE TABLE users ( ... );

9. שיתוף מבנה הטבלאות (Schema) בין חברי צוות
שיטה המקובלת היא שימוש ב-pg_dump והפעלת הקובץ באמצעות psql.
גישה זאת מחולקת למספר שלבים:

א. הוצאת הסכמה (Dump)
יש להוציא את מבנה הטבלאות (סכמה) בלבד, ללא הנתונים בפועל, וללא בעלות (Owner) כדי למנוע שגיאות.
נעבור לתיקייה הרלוונטית במקרה שלנו זה הbackend
עבור לתיקיית הפרויקט: בטרמינל, עבור לתיקיית הפרויקט המשותפת שלך (למשל, BuyForce_project\backend).
ונריץ את הפקודה הבאה:
"C:\Program Files\PostgreSQL\18\bin\pg_dump" -U postgres -s --no-owner -f buyforce_schema.sql buyforce
**--no-owner**: מבטל את שינוי הבעלות, כדי למנוע שגיאות בשרתים שונים.

תוצאה: ייווצר קובץ בשם buyforce_schema.sql המכיל את כל פקודות ה-CREATE TABLE.

ב. שיתוף והפצה:
העלה ל-Git: ודא שקובץ buyforce_schema.sql מוכנס לקונטרול ב-Git.
Commit ו-Push: בצע Commit ו-Push ל-GitHub כדי שחברי הצוות יוכלו לבצע git pull ולקבל את הסכמה המעודכנת.

ג. טעינת הסכמה:
כאשר חבר צוות רוצה לעדכן את מסד הנתונים שלו (או ליצור אותו לראשונה)
יצירת DB ריק: חבר הצוות מוודא שיש לו מסד נתונים ריק בשרת שלו (למשל, buyforce_local)

טעינת הקובץ: הוא מריץ את הפקודה הבאה מהטרמינל (כאשר הוא בתיקיית הקובץ):
psql -U <your user name> -d buyforce_local -f buyforce_schema.sql

-d buyforce_local: שם מסד הנתונים הריק שאליו הוא טוען את הסכמה.

תוצאה: כל הטבלאות שיצרת יופיעו כעת במסד הנתונים שלו, והוא יוכל לראות אותן באמצעות \dt ב-psql או דרך pgAdmin 4.


10. מה נעשה בהמשך:
נשתמש במנגנון ה-API של הפרויקט.

API (Node.js): הקוד בתיקיית ה-backend שלך (למשל, בקובץ index.js) מכיל את נקודות הקצה (Endpoints) שאתם בונים. כל נקודת קצה (כמו POST /api/products) אחראית לבצע פעולה ב-PostgreSQL.
